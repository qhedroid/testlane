Relay — Wireframe Concepts

2026-07-02 · Wireframe fidelity (structure & flow, not visual design) · Companion to ux-critique.md

Four areas from the design brief: A. Reporting & Analytics · B. Export & Reporting · C. Re-Run Management · D. Test Case Organization. Grey boxes = existing patterns reused as-is; dashed borders = new surfaces. Numbered dots are annotated below each frame.

Constraints respected: frontend-only MVP (no server push, no real integrations — anything needing them is marked stub); the three-pane Test Runs execution workspace is untouched — Re-Run concepts hook into the run header, summary card, and runs list only; admin (/admin/*) stays separate.

Replaces the "Planned module" placeholder. One page frame, one control bar that drives every widget, drill-down from any chart to a case-level table. Reuses the dashboard's existing KPI/donut vocabulary so it reads as the same product.

C · Re-Run Management — from any completed/closed run

Tracker: 0/4, all Not Started; only "Duplicate test run" exists. Concept: a re-run is a linked, filtered new run — created from the run header's existing More… menu and from the close-run moment. The three-pane execution UX is not touched; a re-run opens as an ordinary run.
